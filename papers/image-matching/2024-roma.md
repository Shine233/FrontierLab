# RoMa：鲁棒的稠密特征匹配

## 结论先行

- **一句话定位**：RoMa 是稠密（dense）两视图匹配路线在 2024 年的强 SOTA baseline，核心思想是「用冻结的 DINOv2 提供鲁棒的 coarse 特征 + 专用 ConvNet 提供可精定位的 fine 特征」，在最难的跨域基准上提升最大。
- **核心方法**：coarse-to-fine 稠密 warp 回归；冻结 DINOv2 coarse 特征 + VGG19 ConvNet fine 特征；定制 transformer match decoder 预测 anchor probabilities 表达多模态分布；coarse 阶段 regression-by-classification、fine 阶段 robust regression。
- **论文证据**：MegaDepth-1500 AUC@5/10/20 = 62.6/76.7/86.3；ScanNet-1500 = 31.8/53.4/70.9（论文称首次 AUC@20° 超 70）；IMC2022 mAA@10 = 88.0（相对误差约 -26%）；WxBS mAA@10px = 80.1（headline +36%）。
- **代码状态**：GitHub 公开推理 + 训练（`experiments/` + `torchrun`，`--only_test` 评测），权重自动下载；自有代码 MIT，DINOv2 为 Apache-2.0；按仓库约定 `training_open_source: true`。
- **工程判断**：RoMa 精度/鲁棒性明显高于稀疏匹配器（LightGlue/SuperGlue），但稠密 decoder + DINOv2 计算/显存更重；560×560、batch 8、RTX6000 上约 199 ms/对，比 DKM 仅多约 7%。适合做高精度匹配主路线或重建/定位前端，不适合极致低延迟稀疏场景。

## 1. 这篇论文解决什么问题？

### 已确认的论文事实

- **问题定义**：两视图特征匹配在大视角变化、外观变化、跨域（不同天气/相机/场景）时鲁棒性不足。RoMa 想用一个稠密匹配模型同时拿到高精度和强泛化。
- **输入 / 输出**：输入两张图像，输出稠密像素级 warp（对应场）和置信度，下游接 RANSAC/MAGSAC 求相对位姿。
- **目标场景**：相对位姿估计 / two-view geometry、视觉定位、3D 重建前端。
- **与现有方法差异**：相对 LoFTR/ASpanFormer 等 semi-dense detector-free 方法，RoMa 是全稠密；相对 DKM（同作者前作）改用冻结基础模型特征和新 decoder/loss。

## 2. 方法概览

- **核心想法**：coarse-to-fine 稠密匹配。冻结 **DINOv2** 提供鲁棒但粗糙的 coarse 特征，与专门训练的 **ConvNet（VGG19）fine 特征** 结合，构建可精确定位的特征金字塔。
- **模型结构**：定制 transformer match decoder，预测 anchor probabilities，以表达匹配的多模态分布。
- **训练目标**：coarse 阶段用 **regression-by-classification**；fine 阶段用 **robust regression** 回归 warp。
- **推理流程**：coarse 匹配 → 逐级 refine → 输出稠密 warp + certainty → 采样对应点接几何求解器。

## 3. 关键贡献

1. 把冻结基础模型（DINOv2）coarse 特征与专用 ConvNet fine 特征解耦组合，兼顾鲁棒性与定位精度。
2. transformer match decoder + anchor probability，建模多模态匹配分布。
3. regression-by-classification（coarse）+ robust regression（fine）的损失设计。
4. 在 MegaDepth-1500 / ScanNet-1500 / IMC2022 / WxBS 上全面 SOTA，难基准提升最大。

## 5. 局限与风险

### 论文明确承认

- 依赖**有监督对应**，限制可用训练数据量（用冻结基础模型特征部分缓解泛化）。
- 训练稠密匹配是对下游任务的**间接优化**，直接在下游训练可能更优。

### 我推断的风险

- 稠密匹配计算/显存成本高于稀疏匹配器，实际部署成本明显高于 LightGlue。
- 高分辨率 / 大 batch 显存压力需评估（具体显存 to verify）。

### 工程 / 许可证风险

- 自有代码 MIT（商用友好），但依赖 **DINOv2（Apache-2.0）**，DINOv2 权重本身使用条款需单独确认（to verify）。
- 数据准备依赖外部仓库 DKM，复现训练有额外配置成本。

## 6. 与相似方法对比

| Method | 相同点 | 不同点 | 何时选它 |
|---|---|---|---|
| DKM | 同作者，同稠密 warp 回归框架 | RoMa 改用冻结 DINOv2 coarse + 专用 ConvNet fine + anchor decoder + RbC/robust loss，全面超越 DKM | 需要 DKM 升级版选 RoMa |
| LoFTR / ASpanFormer | 都是 detector-free 匹配 | RoMa 全稠密、鲁棒性更强，AUC 全面领先 | 需要最强鲁棒匹配选 RoMa |
| LightGlue / SuperGlue | 都做两视图匹配 | RoMa 精度/鲁棒性更高但更慢更重；稀疏方法更快 | 要精度上限选 RoMa，要速度/低显存选 LightGlue |
| RoMa v2 | 同系后继 | v2 用 DINOv3、Attention 替代 GP、两阶段训练、预测协方差、快 ~1.7× | 困难/跨域稠密匹配优先 v2 |
