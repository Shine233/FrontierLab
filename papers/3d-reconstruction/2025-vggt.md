# VGGT：视觉几何基础 Transformer

## 结论先行
- VGGT 把多视图几何从「成对匹配 + 优化」范式推进到「单次前馈大模型直接回归全部几何量」的范式：一个网络同时输出相机内外参、深度图、点图和 3D 点轨迹，推理约 0.2s（证据：论文 Table 1 前馈版 ~0.2s vs DUSt3R ~7s，运行时间在单张 H100 上测得）。
- 精度上它在无已知相机的设定下超过依赖优化的 DUSt3R/MASt3R：相机位姿 AUC@30 在 CO3Dv2 达 88.2（DUSt3R 76.7），ETH3D 点图 Overall 从 DUSt3R 的 1.005 降到 0.677（证据：Table 1/3）。
- 关键设计是「交替注意力（alternating attention）」——逐帧自注意力与全局跨帧注意力交替堆叠，使模型能处理任意帧数且保持置换对称性（需一帧作参考坐标系）。
- 工程可用性高：训练代码、1B 权重均已开源，权重从 HuggingFace 自动下载；代码许可自 2025-07-29 起已改为商用友好（排除军事用途），但原始 VGGT-1B 权重仍非商用，商用需申请 VGGT-1B-Commercial 权重（推断风险：权重许可审批是落地考量）。
- 定位为「几何基础模型」：预训练 backbone 可直接迁移到非刚性点跟踪、新视角合成等下游任务，并获 CVPR 2025 Best Paper Award，是仓库内后续工作的本体（landmark）。

## 1. 这篇论文解决什么问题？
- 问题定义：给定 1 到数百张同场景图像，一次性推断完整 3D 场景属性——相机内外参、逐像素深度图、点图（point map）、以及跨帧 3D 点轨迹。
- 输入输出：输入 N 张 RGB 图像；输出每帧的相机参数、深度图、点图，以及可查询的 3D 点轨迹。
- 目标场景：静态/准静态多视图重建、SfM 替代、稠密重建、下游几何任务的通用 backbone。
- 与现有方法差异：DUSt3R/MASt3R 依赖成对点图 + 全局对齐优化，帧数增多时开销与误差累积上升；传统 SfM（COLMAP 等）依赖迭代 BA。VGGT 用单次前馈直接给出全局一致几何，去掉后处理优化环节（可选再叠加 BA 进一步提精度）。

## 2. 方法概览
- 核心想法：把多视图几何当作一个可端到端学习的回归问题，用大 Transformer 直接映射「图像集合 → 几何量集合」，而非依赖手工优化管线。
- 模型结构：图像经 DINO 类 patch tokenizer 编码后，进入交替注意力主干——帧内自注意力（frame-wise）与全局跨帧注意力（global）逐层交替；多个预测头分别输出相机参数、深度、点图与跟踪特征。整体约 1B 参数。
- 训练目标：对相机参数、深度图、点图施加回归损失，并对点轨迹做跟踪监督；多任务联合训练。
- 推理流程：前馈单次给出所有几何量；点图既可由专用头直接预测，也可由「预测深度 + 预测相机反投影」得到，后者在 ETH3D 上更优（Depth+Cam Overall 0.677 vs Point 0.709）。可选叠加 BA 进一步提升位姿精度。

## 3. 关键贡献
1. 提出纯前馈的视觉几何基础 Transformer，单模型同时输出相机、深度、点图、3D 轨迹，1 秒内完成重建，无需几何后优化。
2. 交替注意力架构，支持任意帧数（1~数百）输入并在多任务上取得 SOTA。
3. 证明预训练 VGGT 可作为通用几何特征 backbone，提升非刚性点跟踪与新视角合成等下游任务。
4. 在多个基准上以更快速度超过依赖优化的 DUSt3R/MASt3R/VGGSfM，且在无已知相机设定下仍领先。

## 4. 实验与证据
| 维度 | 内容 |
|---|---|
| 数据集 | RealEstate10K、CO3Dv2（位姿）、DTU（多视图深度）、ETH3D（点图）；训练用 TartanAir、ScanNet 等多源数据 |
| Baseline | DUSt3R、MASt3R、VGGSfM v2，以及传统 MVS（多为已知 GT 相机的方法） |
| 指标 | 相机位姿 AUC@30；深度/点图 Accuracy、Completeness、Overall（Chamfer） |
| 主要结果 | 位姿 CO3Dv2 AUC@30 88.2（前馈，~0.2s）/91.8（+BA，~1.8s），DUSt3R 76.7 / MASt3R 81.8 / VGGSfM v2 83.4；Re10K 85.3/93.5 vs DUSt3R 67.7。ETH3D 点图 Overall 0.677（Depth+Cam）vs DUSt3R 1.005、MASt3R 0.826。DTU Overall 0.382（无 GT 相机）vs DUSt3R 1.741（MASt3R 0.374 但用 GT 相机三角化，非同一设定） |
| 消融 | 「预测深度 + 预测相机反投影」得到的点图优于专用点图头（0.677 vs 0.709），作者归因于把复杂任务分解为更简单子问题 |
| 失败案例 | 论文承认（推断）：对高度动态/非刚性场景、极端外推视角、超长序列的一致性仍受限；需一帧作为参考坐标系 |

（注：以上数值来自 arXiv v1 HTML 的 Table 1/2/3，已联网核实；运行时间在单张 H100 上测得，各方法均未在 Re10K 上训练。）

## 5. 局限与风险
- 论文承认：模型规模大（~1B），部署对显存有要求；点图直接头精度略逊于深度+相机反投影组合。
- 推断风险：训练数据以静态/准静态场景为主，强动态与非刚性场景下几何一致性可能退化；超长序列的全局一致性依赖全局注意力，帧数极大时的开销与稳定性需评估。
- 工程落地风险：1B 模型推理需较强 GPU；虽单次前馈快，但显存/吞吐是批量落地考量。
- 许可证风险：代码许可自 2025-07-29 起为商用友好（排除军事用途）；但原始 VGGT-1B 权重仍非商用，商用需申请 VGGT-1B-Commercial 权重（审批流程类似 LLaMA，官方称其性能与原版相当，CO3D AUC@30 90.37 vs 89.98）。

## 6. 与相似方法对比
| Method | 相同点 | 不同点 | 何时选它 |
|---|---|---|---|
| DUSt3R | 前馈回归点图、去 SfM 优化 | DUSt3R 成对处理 + 全局对齐优化，慢且随帧数扩展差；VGGT 单次多帧、多任务、更快更准 | 需要轻量成对场景或作为谱系参照时选 DUSt3R，否则 VGGT |
| MASt3R | 前馈几何 + 匹配 | MASt3R 强化匹配但仍需优化对齐；VGGT 无需后处理且覆盖更多任务 | 侧重稠密匹配/定位时可比较，通用几何优先 VGGT |
| VGGSfM v2 | 面向 SfM 相机估计 | VGGSfM 仍偏优化式管线，速度慢（~10s）；VGGT 前馈 ~0.2s | 需与传统 SfM 兼容管线时选 VGGSfM |
| COLMAP（传统 SfM） | 输出相机 + 稀疏/稠密几何 | COLMAP 迭代 BA、精度稳但慢且对纹理敏感；VGGT 学习式、快、鲁棒但需 GPU | 离线高精度、算力受限或无训练数据时选 COLMAP |

## 7. 复现判断
- Git 地址：https://github.com/facebookresearch/vggt
- 是否开源：是（代码 + 1B 权重）。
- 是否开源训练：是，2025-07-06 起 `training/` 目录含训练代码及自定义数据集微调示例。
- 代码/权重/数据可用性：权重 `facebook/VGGT-1B` 从 HuggingFace 自动下载；另有 `VGGT-1B-Commercial`（需申请，商用）。训练数据为多源公开数据集组合，完整复现需自行准备。
- 预计成本：推理单卡可跑（1B 模型，需较大显存；Ampere+ 用 bfloat16 autocast）；从头训练成本高（大模型 + 多源数据），一般直接用官方权重。
- 最小复现路径：clone 仓库 → 安装依赖 → 运行 demo（Gradio/Viser），权重自动下载 → 在自有多视图图像上前馈推理，查看相机/深度/点图输出。
- 是否值得复现：推理级复现值得（几何基础模型、下游可迁移）；从头训练非必要，除非做架构改动或数据实验。商用务必使用 VGGT-1B-Commercial 权重并遵守代码许可。
