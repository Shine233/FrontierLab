# DINOv3

## 结论先行

- **一句话定位**：DINOv3 是 DINO 家族从“通用 frozen backbone”走向“更大规模、更强 dense features、更高分辨率和多部署形态”的版本。
- **核心问题**：DINOv2 继续扩大模型和训练步数时，全局分类表征会继续变强，但 patch-level dense features 可能退化；DINOv3 的关键创新是用 **Gram anchoring** 稳住 patch 间相似关系。
- **核心方法**：在 DINOv2-style DINO+iBOT+KoLeo recipe 上扩展数据和模型，训练 ViT-7B/16；检测到 dense feature degradation 后，用早期 dense 表现更好的 Gram teacher 约束 student patch Gram matrix；再做高分辨率适配、多尺寸蒸馏和文本对齐。
- **规模变化**：论文报告 web 图像 curated set LVD-1689M、卫星数据 SAT-493M；主模型约 6.7B 参数，远大于 DINOv2 ViT-g 约 1.1B。
- **开源状态**：GitHub 公开，使用自定义 DINOv3 License；提供训练/评测/蒸馏相关代码和 configs，权重通过 Meta 下载/Hugging Face 提供；训练代码开源，但旗舰模型预训练用私有数据，完整数据复刻不可行。
- **对本仓库价值**：如果后续研究 dense foundation features、3D correspondence、segmentation/depth、遥感/自动驾驶视觉 backbone，DINOv3 应作为高优先级 baseline。

## 1. 这篇论文解决什么问题？

DINOv2 已经证明自监督视觉特征可以很通用。但当模型更大、训练更久时出现一个矛盾：

- `[CLS]` / 全局特征继续变强，分类、检索、OOD transfer 可能继续提升；
- patch token / dense feature 却可能变差，表现为分割、深度、tracking、correspondence 下降。

DINOv3 的问题定义就是：**如何在继续 scaling 自监督视觉模型时，不牺牲 dense feature 的局部一致性？**

## 2. 方法概览

### 2.1 给初学者的直觉

想象一个学生越学越会“判断整张图是什么”，但慢慢不再关心“物体边界在哪里、每个局部和哪个局部相似”。这就是 dense feature degradation。

DINOv3 的解决方法像是给学生加一条规则：

> 你可以更新每个 patch 的具体表示，但不能破坏 patch 之间“谁和谁相似”的关系结构。

这个“关系结构”就是 Gram matrix：把一张图里所有 patch 两两做 dot product，得到一个 patch-patch 相似度矩阵。

### 2.2 Gram anchoring

| 概念 | 解释 | 为什么有用 |
|---|---|---|
| Patch features | ViT 每个图像 patch 的输出 token | dense tasks 依赖它们，而不是只依赖 `[CLS]` |
| Gram matrix | patch features 两两相似度矩阵 | 描述局部结构和区域关系 |
| Gram teacher | 训练早期 dense 表现更好的 teacher checkpoint | 早期局部一致性好，用来当锚点 |
| Gram loss | 让 student 的 Gram matrix 接近 Gram teacher | 保持局部相似结构，不强行固定每个 feature 方向 |
| High-res Gram | teacher 用更高分辨率提特征再下采样 | 给 student 更细、更平滑的 dense 结构信号 |

与直接蒸馏 feature 不同，Gram anchoring 不要求 student 每个 patch vector 完全复制 teacher，只要求 patch 之间的相似关系别坏掉。因此它更适合在保持 global feature 上升的同时修复 dense feature。

### 2.3 训练/后处理 pipeline

1. **数据扩展**：构建 LVD-1689M web image curated set，并训练卫星分支 SAT-493M。
2. **模型扩展**：训练 ViT-7B/16 级模型，引入现代 ViT 设计如 axial RoPE、register tokens 等。
3. **基础 SSL 目标**：沿用 DINO global loss、iBOT patch loss、KoLeo、Sinkhorn-Knopp 等 recipe。
4. **Gram anchoring refinement**：选择 dense 表现好的早期 teacher，用 Gram loss 修复/保持 patch-level consistency。
5. **High-resolution adaptation**：让模型在更高输入分辨率下仍保留稳定 dense features。
6. **Model distillation**：把 7B teacher 蒸馏到 ViT-S/B/L/H+、ConvNeXt 等多尺寸模型。
7. **Text alignment**：冻结视觉 backbone，对齐文本 encoder，支持 dino.txt 零样本/开放词表能力。

## 3. 关键贡献

1. **识别并解决 dense feature degradation**：指出大规模长训练中 global 和 dense 表现可能分离，并用 Gram anchoring 修复。
2. **更大规模自监督视觉训练**：将 DINOv2-style recipe 扩展到 LVD-1689M 和 ViT-7B/16。
3. **强 dense foundation features**：重点验证分割、深度、tracking、3D correspondence、object discovery 等 patch-level 任务。
4. **多形态模型族**：ViT、ConvNeXt、web/satellite、text-aligned、CHMv2 等生态，覆盖研究与部署。
5. **公开训练/评测代码**：repo 包含 `dinov3/train`、`dinov3/eval`、configs、multi-distillation 等；但旗舰训练数据私有。

## 4. 实验与证据

| 维度 | 内容 |
|---|---|
| 数据集 | LVD-1689M web 图像；SAT-493M 卫星图像；ADE20K、Cityscapes、VOC、NYUv2、KITTI、COCO、DAVIS 等 |
| Baseline | DINOv2 with registers、SigLIP 2、PE/PEspatial、SAM/PEspatial、AM-RADIO、Franca、Web-DINO 等 |
| 指标 | ImageNet linear top-1、ADE/Cityscapes/VOC mIoU、NYUv2/KITTI RMSE、detection AP、tracking/correspondence 指标 |
| 主要结果 | 论文报告 DINOv3 frozen dense features 在多类 dense benchmarks 上显著超过前代和强公开 backbones；高分辨率适配提升 dense 表现 |
| 消融 | 未加 Gram anchoring 时长训 dense 表现下降；Gram teacher iteration 和高分辨率 Gram 影响显著 |
| 失败案例 | exact flagship training 依赖私有数据与超大算力；自定义 license 和权重申请流程需审查 |

## 5. 局限与风险

- **论文/仓库明确可见限制**：ViT-7B/16 exact setup 使用私有数据；README 给出训练命令/config，但 `<DATASET>` 需要用户自行替换。
- **我推断的风险**：DINOv3 作为 frozen backbone 很强，但未必自动解决所有下游任务；小数据/强领域偏差任务仍可能需要 adapter、linear probe 或微调。
- **工程落地风险**：7B teacher 训练需要 256 GPU 级别 H100/SLURM 环境；权重下载和许可不是简单 Apache 模式；部分 weights 需申请。
- **许可证 / 数据风险**：DINOv3 License 是自定义协议；商用、再分发、模型衍生使用需逐条审查。

## 6. 与相似方法对比

| Method | 相同点 | 不同点 | 何时选它 |
|---|---|---|---|
| DINO v1 | self-distillation + ViT | v1 是机制发现，v3 是大规模 dense foundation model | 学原理从 v1 开始；做高上限 dense backbone 选 v3 |
| DINOv2 | 同样 DINO+iBOT/KoLeo 风格，自监督通用 backbone | v3 扩到更大数据/模型，并用 Gram anchoring 解决 dense 退化 | 常规 frozen backbone 可先用 v2；dense/high-res/遥感优先 v3 |
| CLIP/SigLIP | 都可做通用视觉编码器 | CLIP/SigLIP 强图文语义和 zero-shot；DINOv3 强局部 dense feature | 文字检索/开放词表选 CLIP/SigLIP；分割/深度/对应/跟踪选 DINOv3 |
| SAM/PEspatial | 都提供 dense/空间特征 | SAM 是分割 prompt 模型，DINOv3 是通用自监督 backbone | 需要交互式 mask 选 SAM；需要可迁移 patch representation 选 DINOv3 |

## 7. 复现判断

- Git 地址：<https://github.com/facebookresearch/dinov3>
- 是否开源：是，仓库公开；许可证为自定义 DINOv3 License，需审查使用限制。
- 是否开源训练：是。仓库包含 `dinov3/train/train.py`、`dinov3/configs/train/*`、multi-distillation、eval configs；但 exact 旗舰模型训练数据为私有，完整数据不可复刻。
- 代码可用性：训练、推理、线性评测、分割/深度/文本对齐等代码持续更新。
- 权重可用性：Meta 下载页、Hugging Face collection、Transformers/timm 支持；部分权重需申请链接。
- 数据可获得性：ImageNet/公开下游可复现；LVD-1689M / SAT-493M 完整预训练数据不可直接复刻。
- 预计环境成本：fast setup 是 4 个 H100-80GB 节点（32 GPU）约 14 小时；exact ViT-7B/16 setup 是 32 节点（256 GPU）级别。
- 最小复现路径：先用 Hugging Face / torch.hub 加载 `facebook/dinov3-*` 小模型抽 patch features；跑 ADE20K/NYUv2 线性 probe 或 PCA 可视化；训练只建议从 ImageNet-1k fast config 或小规模蒸馏开始。
- 是否值得复现：值得做 inference/linear-probe 层复现和作为 dense backbone baseline；不建议从零复刻 7B pretraining。
